# 04 — API Design

## Principles

- RESTful resource-oriented design
- All responses in JSON
- API versioning via URL prefix: `/api/v1/`
- Consistent error envelope across all endpoints
- OpenAPI 3.1 spec auto-generated by FastAPI
- Authentication via `Authorization: Bearer <JWT>` header on all protected endpoints

---

## Standard Response Envelopes

### Success
```json
{
  "data": { ... },
  "meta": {
    "request_id": "uuid",
    "timestamp": "ISO8601"
  }
}
```

### Error
```json
{
  "error": {
    "code": "PLAN_UPGRADE_REQUIRED",
    "message": "This feature requires a Pro subscription.",
    "details": {
      "required_plan": "pro",
      "upgrade_url": "https://app.example.com/upgrade"
    }
  },
  "meta": {
    "request_id": "uuid",
    "timestamp": "ISO8601"
  }
}
```

### Standard Error Codes
| HTTP | Code | Meaning |
|---|---|---|
| 400 | `VALIDATION_ERROR` | Pydantic validation failed |
| 400 | `INVALID_HS_CODE` | HS code not found in dataset |
| 401 | `UNAUTHENTICATED` | Missing or invalid JWT |
| 403 | `PLAN_UPGRADE_REQUIRED` | Feature requires Pro |
| 403 | `FORBIDDEN` | Authenticated but not authorized |
| 404 | `NOT_FOUND` | Resource not found |
| 409 | `CONFLICT` | Duplicate resource |
| 422 | `CALCULATION_ERROR` | Engine failed to compute |
| 429 | `RATE_LIMIT_EXCEEDED` | Too many requests |
| 500 | `INTERNAL_ERROR` | Unexpected server error |
| 503 | `DATA_STALE` | Tariff data is older than threshold |

## Endpoints

### Auth

#### `POST /api/v1/auth/google`
Exchange Google ID token for application JWT.

**Request:**
```json
{
  "id_token": "google_id_token_string"
}
```

**Response 200:**
```json
{
  "data": {
    "access_token": "jwt_string",
    "token_type": "bearer",
    "expires_in": 3600,
    "user": {
      "id": "uuid",
      "email": "user@example.com",
      "display_name": "Jane Smith",
      "plan": "free"
    }
  }
}
```

#### `POST /api/v1/auth/refresh`
Refresh an access token using a refresh token.

#### `DELETE /api/v1/auth/session`
Logout — invalidate refresh token.

### Subscriptions

#### `POST /api/v1/subscriptions/checkout`
Create Stripe Checkout session for Pro upgrade.

**Request:**
```json
{
  "plan": "pro",
  "billing_period": "monthly"
}
```

**Response 200:**
```json
{
  "data": {
    "checkout_url": "https://checkout.stripe.com/..."
  }
}
```

#### `POST /api/v1/subscriptions/portal`
Get Stripe Customer Portal link for subscription management.

#### `POST /api/v1/webhooks/stripe`
Stripe webhook receiver. Validates Stripe signature. Not authenticated by JWT.

### Tariff Lookup (Free)

#### `GET /api/v1/tariff/hs-codes/search`
Search HS codes by keyword or code prefix.

**Query params:** `q` (string), `jurisdiction` (UK|EU), `limit` (default 10, max 50)

**Response 200:**
```json
{
  "data": {
    "results": [
      {
        "code": "8471300000",
        "description": "Portable automatic data processing machines...",
        "level": 10,
        "supplementary_unit": "p/st"
      }
    ],
    "total": 3
  }
}
```

#### `GET /api/v1/tariff/hs-codes/{code}`
Get details for a specific HS code including applicable measures.

**Query params:** `jurisdiction` (UK|EU)

**Response 200:**
```json
{
  "data": {
    "code": "8471300000",
    "description": "...",
    "measures": [
      {
        "measure_type": "AD_VALOREM",
        "rate_ad_valorem": 0.0,
        "country_of_origin": null,
        "valid_from": "2024-01-01",
        "valid_to": null
      }
    ],
    "supplementary_unit": "p/st"
  }
}
```

### Calculations (Core)

#### `POST /api/v1/calculations/sync`
Run a synchronous calculation. Best for simple, single-line shipments.

**Plan gate:** Partial (free users get restricted engine set automatically)

**Request:**
```json
{
  "shipment": {
    "incoterm": "FOB",
    "origin_country": "CN",
    "destination_country": "GB",
    "port_of_entry": "GBFXT",
    "freight_cost": { "amount": "1500.00", "currency": "GBP" },
    "insurance_cost": { "amount": "50.00", "currency": "GBP" },
    "handling_cost": { "amount": "0.00", "currency": "GBP" }
  },
  "lines": [
    {
      "line_number": 1,
      "hs_code": "8471300000",
      "description": "Laptop computers",
      "invoice_value": { "amount": "10000.00", "currency": "USD" },
      "quantity": 10,
      "quantity_unit": "units",
      "gross_weight_kg": "25.5",
      "country_of_origin": "CN",
      "has_proof_of_origin": false,
      "royalties": { "amount": "0", "currency": "GBP" },
      "assists": { "amount": "0", "currency": "GBP" },
      "buying_commission": { "amount": "0", "currency": "GBP" },
      "selling_commission": { "amount": "0", "currency": "GBP" },
      "is_related_party": false
    }
  ],
  "options": {
    "jurisdiction": "UK",
    "calculation_date": "2026-02-23",
    "include_clearance_estimate": true
  }
}
```

**Response 200:**
```json
{
  "data": {
    "request_id": "uuid",
    "status": "complete",
    "confidence_score": 0.91,
    "totals": {
      "customs_value": { "amount": "9250.00", "currency": "GBP" },
      "total_duty": { "amount": "0.00", "currency": "GBP" },
      "total_vat": { "amount": "1850.00", "currency": "GBP" },
      "total_excise": { "amount": "0.00", "currency": "GBP" },
      "total_clearance": { "amount": "250.00", "currency": "GBP" },
      "total_landed_cost": { "amount": "11350.00", "currency": "GBP" }
    },
    "line_results": [ { "line_number": 1, "..." : "..." } ],
    "warnings": [],
    "audit_trail_available": true,
    "engines_used": ["classification", "customs_valuation", "tariff_measure", "vat", "fx"]
  }
}
```

#### `POST /api/v1/calculations/async`
Submit a calculation for background processing. Use for multi-line invoices.

**Plan gate:** PRO only for multi-line

**Response 202:**
```json
{
  "data": {
    "task_id": "uuid",
    "status": "pending",
    "poll_url": "/api/v1/calculations/uuid/status"
  }
}
```

#### `GET /api/v1/calculations/{task_id}/status`
Poll calculation status.

**Response 200:**
```json
{
  "data": {
    "task_id": "uuid",
    "status": "processing",
    "progress": { "lines_processed": 12, "lines_total": 45 }
  }
}
```

#### `GET /api/v1/calculations/{request_id}/result`
Retrieve completed calculation result.

**Response 200:** Full CalculationResult object

#### `GET /api/v1/calculations/{request_id}/audit`
Retrieve the full audit trail for a completed calculation.
**Plan gate:** PRO only

**Response 200:**
```json
{
  "data": {
    "steps": [
      {
        "sequence": 1,
        "engine": "customs_valuation",
        "step_name": "incoterm_gap_adjustment",
        "formula_description": "FOB origin: added freight GBP 1,500 + insurance GBP 50 to reach CIF border value",
        "input_snapshot": { "..." : "..." },
        "output_snapshot": { "customs_value": "9250.00" }
      }
    ]
  }
}
```

### Calculation History

**Query params:** `limit`, `offset`, `status`, `from_date`, `to_date`

#### `DELETE /api/v1/calculations/{request_id}`

## Rate Limiting

| Endpoint Group | Free | Pro |
|---|---|---|
| `POST /calculations/*` | 10/hour | 500/hour |
| `GET /tariff/*` | 100/hour | 1000/hour |
| `POST /auth/*` | 20/hour | 20/hour |

Rate limits tracked in Redis. Headers returned: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`.

## Pagination

All list endpoints use offset pagination:

```json
{
  "data": { "results": [...] },
  "meta": {
    "total": 142,
    "limit": 20,
    "offset": 40,
    "has_more": true
  }
}
```
